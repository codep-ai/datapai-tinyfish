# DataP.ai × TinyFish — Database

PostgreSQL schema shared by two services on the same EC2 host:

| Service | Stack | Role |
|---------|-------|------|
| `datapai-tinyfish` (port 3085) | Next.js 16 + node-postgres | Scan pipeline, auth, watchlist, TA cache reads |
| `datapai-streamlit` (port 8000) | Python FastAPI + psycopg2 | AI agents, TA signal writes, stock chat |

## Tables

### Group 1 — Scan Pipeline (Next.js owned)

| Table | Description |
|-------|-------------|
| `companies` | Monitored ticker universe; seeded from `lib/universe.ts` |
| `runs` | One row per full scan pipeline run |
| `snapshots` | Raw + cleaned IR page text per ticker per run |
| `diffs` | Text diff (similarity, added/removed lines, snippet) between two snapshots |
| `analyses` | AI signal analysis result; joins to snapshot + diff |
| `scan_events` | Step-level execution log for real-time progress streaming |
| `prices` | EOD price cache (Yahoo Finance / Polygon.io) |
| `stock_directory` | Searchable stock universe for on-demand ticker lookup |

### Group 2 — TA Signal & Chart Cache (Python writes, both read)

| Table | Description |
|-------|-------------|
| `ta_signals` | Cached TA signal per ticker: RSI, MACD, Bollinger, trend, price. Generated by Python `technical_analysis.py`. TTL ~8 hours. |
| `chart_analyses` | Cached AI chart vision analysis (base64 chart PNG + markdown). Generated by Python. TTL ~24 hours. |

### Group 3 — Auth & Watchlist (Next.js owned)

| Table | Description |
|-------|-------------|
| `users` | Next.js auth users. `id` is text UUID. Password is PBKDF2-SHA512. |
| `sessions` | Server-side session tokens with 30-day TTL |
| `watchlist` | Per-user stock watchlist (`user_id` → `users.id`) |

### Group 4 — AI Stock Chat (Python owned)

| Table | Description |
|-------|-------------|
| `chat_sessions` | One conversation thread per `(user_id, ticker)`. `user_id` is integer (Python auth). |
| `chat_messages` | Individual messages in a chat session (role: user/assistant) |
| `ticker_context_cache` | Short-lived RAG context cache keyed by `(ticker, context_type)`. Types: `ir_snapshot`, `ta_signal`. |
| `user_profiles` | Per-user chat preferences: risk tolerance, portfolio tickers, language |

> **Note on user systems:** Groups 3 and 4 have separate user tables. Next.js uses `users.id` (text UUID). Python chat uses integer `user_id` in `chat_sessions` / `user_profiles`. They do not cross-reference.

## Key Indexes

```sql
-- Fast per-ticker lookups (most common query pattern)
idx_snap_ticker_fetched       ON snapshots(ticker, fetched_at DESC)
idx_analyses_snap             ON analyses(snapshot_new_id)
idx_ta_signals_ticker         ON ta_signals(ticker, generated_at DESC)
idx_chart_analyses_ticker     ON chart_analyses(ticker, generated_at DESC)

-- Scan pipeline
idx_scan_events_run           ON scan_events(run_id, ticker, created_at)
idx_stock_dir_symbol          ON stock_directory(symbol)

-- Auth & watchlist
idx_sessions_user             ON sessions(user_id)
idx_watchlist_user            ON watchlist(user_id)

-- Chat
idx_chat_sessions_user_ticker ON chat_sessions(user_id, ticker)
idx_chat_messages_session     ON chat_messages(session_id, created_at)
```

## Useful Queries

```sql
-- Latest signal per ticker
SELECT s.ticker, a.agent_signal_type, a.alert_score, s.fetched_at
FROM datapai.analyses a
JOIN datapai.snapshots s ON a.snapshot_new_id = s.id
ORDER BY s.fetched_at DESC LIMIT 20;

-- Recent TA signals
SELECT ticker, exchange, current_price, rsi, trend, generated_at
FROM datapai.ta_signals
ORDER BY generated_at DESC LIMIT 20;

-- Watchlist count per user
SELECT user_id, COUNT(*) FROM datapai.watchlist GROUP BY user_id;

-- Context cache freshness
SELECT ticker, context_type, created_at, expires_at
FROM datapai.ticker_context_cache
ORDER BY created_at DESC LIMIT 20;

-- Run history
SELECT id, status, started_at, finished_at,
       scanned_count, changed_count, failed_count
FROM datapai.runs ORDER BY started_at DESC LIMIT 10;
```
